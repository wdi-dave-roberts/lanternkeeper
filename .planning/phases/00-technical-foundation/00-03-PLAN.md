Dave must perform these steps in the Godot 4.6.2 editor. Claude cannot create project.godot (per D-10).

**Step 1: Install Godot 4.6.2 (if not already installed)**
- Download standard build (NOT .NET/C#) from https://godotengine.org/download
- macOS: drag to /Applications

**Step 2: Create the project**
- Open Godot, click "New Project"
- Name: "Lanternkeeper" (no spaces)
- Path: point to the existing repo directory `/Users/dave/github/wdi-dave-roberts/lanternkeeper`
- Renderer: Compatibility (OpenGL)
- Click "Create & Edit"

**Step 3: Configure project settings (Project > Project Settings)**

| Setting Path | Value |
|---|---|
| Display > Window > Size > Viewport Width | `1080` |
| Display > Window > Size > Viewport Height | `1920` |
| Display > Window > Stretch > Mode | `canvas_items` |
| Display > Window > Stretch > Aspect | `keep_width` (NOT keep_height, per D-12) |
| Display > Window > Handheld > Orientation | `portrait` |
| Rendering > Renderer > Rendering Method | `gl_compatibility` (should already be set) |
| Input Devices > Pointing > Emulate Touch From Mouse | `true` (FOUND-10) |

**Step 4: Register autoloads (Project > Project Settings > Autoload tab)**

Register in this exact order (top to bottom):
1. `GameState` -- path: `res://autoloads/game_state.gd`
2. `AudioManager` -- path: `res://autoloads/audio_manager.gd`
3. `SceneTransition` -- path: `res://autoloads/scene_transition.gd`
4. `DialogueManager` -- path: `res://autoloads/dialogue_manager.gd`

NOTE: SceneTransition needs a .tscn scene file, not just the .gd. Before registering, create the scene:
- Scene > New Scene > Other Node > CanvasLayer
- Set CanvasLayer layer property to `128`
- Add child: ColorRect
- Set ColorRect layout to "Full Rect" (Layout menu > Full Rect)
- Set ColorRect color to Black
- Attach `autoloads/scene_transition.gd` as the script on the CanvasLayer
- Save as `autoloads/scene_transition.tscn`
- Register `res://autoloads/scene_transition.tscn` (not .gd) as the SceneTransition autoload

Similarly, AudioManager creates nodes programmatically so it can stay as .gd only.

**Step 5: Create audio bus layout**
- Bottom panel > Audio tab
- Add bus "Music" (click Add Bus)
- Add bus "SFX"
- Add bus "Ambient"
- Save (happens automatically as default_bus_layout.tres)

**Step 6: Create test scene (per D-04)**
- Scene > New Scene > Other Node > MarginContainer (or Node2D as root, then add children)
- Better approach: Root = SafeAreaContainer (use shared/ui/safe_area_container.gd)
  - Create root MarginContainer, attach `shared/ui/safe_area_container.gd`
  - Set layout to Full Rect
  - Add child: ColorRect (set color to a calm dark blue, e.g., #1a1a2e)
  - Set ColorRect layout to Full Rect
  - Add child: Label
  - Set Label text to "Lanternkeeper"
  - Center the Label (Layout > Center)
  - Set font size to ~48
- Save as `scenes/main/main.tscn`
- Set as main scene: Project > Project Settings > Application > Run > Main Scene = `res://scenes/main/main.tscn`

**Step 7: Test run**
- Press F5 (or Play button) to run
- Expected: Window appears in portrait (tall) orientation showing dark blue background with "Lanternkeeper" text
- Check Output panel for errors from autoloads loading

```gdscript
func _ready() -> void:
    # Verify autoloads are accessible
    print("=== Lanternkeeper Autoload Verification ===")

    # Test GameState
    var return_count: int = GameState.get_value("return_count", 0)
    GameState.set_value("return_count", return_count + 1)
    print("GameState: return_count = %d (incremented to %d)" % [return_count, return_count + 1])

    # Test DialogueManager
    DialogueManager.load_region("lantern_clearing")
    DialogueManager.line_ready.connect(_on_dialogue_line)
    DialogueManager.get_line("calm")

    # Test AudioManager (just verify it exists, no audio files yet)
    print("AudioManager: ready, buses configured")

    # Test SceneTransition (just verify it exists, don't actually transition)
    print("SceneTransition: ready")

    print("=== All autoloads verified ===")

func _on_dialogue_line(text: String) -> void:

```

This script runs on startup and prints verification to the Output panel. It tests GameState save/load, DialogueManager JSON loading, and confirms AudioManager and SceneTransition are accessible.
