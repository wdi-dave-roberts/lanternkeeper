**Create docs/learning/autoloads-explained.md:**

Write a new page explaining autoloads for someone who has never used Godot. Structure:

1. **What's an Autoload?** -- Plain-language explanation: "An autoload is a script that Godot loads once when the game starts and keeps alive the entire time. Think of them like background services -- they're always running, any scene can talk to them, and they don't disappear when you switch between screens." No jargon. No "singleton pattern" terminology.

2. **Why Lanternkeeper Uses Them** -- Brief explanation: instead of each scene managing its own music, saves, etc., these services handle cross-cutting concerns so scenes stay simple.

3. **The Four Autoloads** -- One section per autoload with:
   - **GameState** -- "Remembers everything about the player's progress. Which scenes are unlocked, how many times they've returned, their last emotional check-in. Saves to a file on the device so progress survives closing the app."
   - **AudioManager** -- "Controls all sound: background music, sound effects, ambient atmosphere. Has three separate volume channels (buses) so you can adjust music without affecting sound effects. Smoothly crossfades between songs instead of abruptly cutting."
   - **SceneTransition** -- "Handles the fade-to-black effect when moving between scenes. Without this, scene changes would be an ugly instant cut. The fade overlay lives here (not in each scene) so it's always available."
   - **DialogueManager** -- "Loads Aetherling's dialogue from JSON files that you (Allie) edit directly. Picks lines based on the player's current emotion. You'll write the dialogue content in JSON files -- no GDScript needed."

4. **How Scenes Talk to Autoloads** -- One simple example: `GameState.get_value("return_count", 0)` -- "This asks GameState how many times the player has returned. The `0` is a fallback value if no data exists yet."

5. **What You Don't Need to Know Yet** -- Reassurance: "You don't need to understand the GDScript inside these files to work on Lanternkeeper. Your job is dialogue and creative direction. These autoloads are Dave's plumbing -- they make your content work."

Tone: warm, direct, no condescension. Written for an intelligent adult who is new to game development, not for a CS student.
