## Key Architecture Notes

Autoload initialization order (Claude's discretion, dependency-analyzed):
1. GameState -- no dependencies; everything else may read state
2. AudioManager -- no dependencies; SceneTransition may trigger audio
3. SceneTransition -- may use AudioManager; must be ready before first scene
4. DialogueManager -- depends only on filesystem; loads lazily

CRITICAL PITFALL: Tweening AudioStreamPlayer volume_db directly in an autoload breaks all audio. Use `create_tween().bind_node(player)` + `tween_method()` instead of `tween_property()`.

CRITICAL RULE: Never call another autoload from `_init()`. Use `_ready()`.
</context>

**autoloads/game_state.gd:**
- `extends Node`
- `const SAVE_PATH := "user://gamestate.dat"` and `const SCHEMA_VERSION := 1`
- `var _state: Dictionary = {}`
- `_ready()` calls `load_state()`
- `load_state()`: checks `FileAccess.file_exists(SAVE_PATH)`, opens with `FileAccess.READ`, calls `file.get_var(true)`, validates `_state.get("schema_version", 0) != SCHEMA_VERSION` and resets to default if mismatch
- `save_state()`: opens with `FileAccess.WRITE`, calls `file.store_var(_state, true)`
- `get_value(key: String, default: Variant = null) -> Variant`: returns `_state.get(key, default)`
- `set_value(key: String, value: Variant) -> void`: sets key, calls `save_state()`
- `_default_state() -> Dictionary`: returns `{"schema_version": SCHEMA_VERSION, "unlocked_scenes": ["lantern_clearing"], "return_count": 0, "last_emotion": ""}`
- All variables and functions have type hints

**autoloads/scene_transition.gd:**
- `extends CanvasLayer`
- `signal transition_finished`
- `@onready var _overlay: ColorRect = $ColorRect`
- `_ready()`: sets `_overlay.color = Color.BLACK` and `_overlay.modulate.a = 0.0`
- `transition_to(scene_path: String) -> void`: creates tween, tweens `_overlay` modulate:a to 1.0 over 0.4s, awaits, calls `get_tree().change_scene_to_file(scene_path)`, creates second tween to fade back to 0.0, awaits, emits `transition_finished`
- NOTE: This file is the .gd half of a .tscn+.gd pair. The .tscn (SceneTransition CanvasLayer layer=128 > ColorRect full-screen) will be created in Plan 03 when Godot editor is available. Add a comment at the top: `# Requires scene_transition.tscn (CanvasLayer > ColorRect) -- created in Godot editor`

**autoloads/audio_manager.gd:**
- `extends Node`
- Constants: `const CROSSFADE_TIME := 1.0`, `const SFX_POOL_SIZE := 8`
- Variables: `var _music_player_a: AudioStreamPlayer`, `var _music_player_b: AudioStreamPlayer`, `var _active_player: AudioStreamPlayer`, `var _sfx_pool: Array[AudioStreamPlayer] = []`, `var _sfx_index: int = 0`
- `_ready()`: create AudioStreamPlayer nodes programmatically (not from .tscn children), assign bus names "Music", "SFX", "Ambient" respectively, add as children, create SFX pool of SFX_POOL_SIZE players on "SFX" bus, set `_active_player = _music_player_a`. Add comment: `# Audio buses (Music, SFX, Ambient) must be created in Godot Audio Bus Layout editor`
- `play_music(stream: AudioStream) -> void`: if `_active_player.playing` and `_active_player.stream == stream` return early. Call `_crossfade_to(stream)`
- `_crossfade_to(new_stream: AudioStream) -> void`: set inactive player's stream, play it, create tween with `create_tween().bind_node(_active_player)` (CRITICAL: bind_node, not bare tween), use `tween_method(_set_volume.bind(_active_player), 1.0, 0.0, CROSSFADE_TIME)` for fade out, parallel `tween_method(_set_volume.bind(inactive_player), 0.0, 1.0, CROSSFADE_TIME)` for fade in, await finished, stop old player, swap active reference
- `_set_volume(value: float, player: AudioStreamPlayer) -> void`: `player.volume_db = linear_to_db(value)`
- `play_sfx(stream: AudioStream) -> void`: round-robin through SFX pool, set stream, play
- `play_ambient(stream: AudioStream) -> void`: placeholder -- create/reuse an ambient player on "Ambient" bus, play the stream
- `set_bus_volume(bus_name: String, volume_linear: float) -> void`: `AudioServer.set_bus_volume_db(AudioServer.get_bus_index(bus_name), linear_to_db(volume_linear))`
- All variables and functions have type hints
