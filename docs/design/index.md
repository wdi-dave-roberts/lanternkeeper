# Game Design

This is the creative home base for Lanternkeeper. Everything here defines what the game *feels* like, what it *says*, and what the player *does* — before any code gets written.

## What's here

**Original Vision**
:   The emotional and mechanical heart of the game. This is the north star — when we're unsure about a decision, we come back here. Allie wrote this.

**Game Parameters**
:   The design bible. Art direction, sound design, session pacing, progression curves, dialogue tone, quest definitions. This is where creative decisions get specific enough to build from.

**Dialogue Guide**
:   How to write and edit Aetherling's lines. Dialogue lives in JSON files that don't require any code knowledge — open them in a text editor, change the words, save. The guide explains the structure and rules.

**Quest Guide**
:   How micro-quests map to the player's emotional state. Each emotion unlocks different gentle prompts. This guide covers the format and how to add new quests.

---

The split is intentional: **Game Parameters** defines the *rules* (how long is a session, what art style, what music feels right), while the **content guides** are the *tools* for writing the actual words and quests that live inside those rules. Parameters are the container. Content fills it.
