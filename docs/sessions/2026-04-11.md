# 2026-04-11 — First Session with Allie

**Duration:** 90-120 minutes
**Goal:** Orient Allie to the project, get her environment running, ship her first change, set a working cadence.

---

## 1. The Why (10 min)

- Walk through the **doc site homepage** together — Allie pulls it up on her PC
- Read her **Original Vision** doc back to her — she wrote it, but seeing it published on a real site lands differently
- Show the full site nav so she knows everything has a home

## 2. Tour the Docs (15 min)

- **Game Design tab** — her home base. Walk through the Overview page, then show Original Vision, Game Parameters, Dialogue Guide, Quest Guide. She owns this entire tab.
- **Architecture tab** — Dave's plumbing. Quick peek so she knows it exists, no deep dive.
- **Reference tab** — show the sub-sections:
    - *Learning Godot* — the 4-step progression she'll work through over time
    - *How We Work* — roles, GSD workflow basics
    - *Quick Lookup* — Git Basics (with examples), Asset Conventions, Glossary. The "look it up when you need it" shelf.

## 3. Environment Setup on Her PC (25-30 min)

Follow **Getting Started** together, screen-sharing:

- [ ] Install Godot 4.6.2 (direct download for Windows — no Homebrew)
- [ ] Install Git + Git LFS (Git for Windows from `git-scm.com`)
- [ ] Clone the repo
- [ ] Open `project.godot` in Godot
- [ ] Orient in the editor: Scene Tree, Inspector, Viewport, Bottom Panel
- [ ] Hit F5 to run — seeing something on screen is the dopamine hit

Don't go through the full 9-step Project Setup guide — that's for building from scratch, which is already done.

## 4. First Real Work Together (20-25 min)

- Open a dialogue JSON file in `data/dialogue/` in a text editor on her machine
- Have her **edit a line of Aetherling dialogue** and save it
- Run the game again to see her change appear
- Walk through the git cycle: `git add`, `git commit`, `git push` — she sees her name in the commit history
- This is the "I just shipped something" moment. Protect it.

## 5. GSD & Claude Code Quick Intro (10 min)

- Show Claude Code running in the terminal — she doesn't need to install it yet
- Explain GSD in one sentence: "It's how we plan, build, and verify work in phases"
- Point her to GSD Workflow: The Basics (under Reference > How We Work) as homework, don't lecture through it
- Show `.planning/ROADMAP.md` so she sees the phases ahead

## 6. Next Steps & Cadence (10 min)

**Her homework before next session:**

- [ ] Read through GDScript Basics (Reference > Learning Godot) — it's a reference, not a test
- [ ] Read the Dialogue Guide (Game Design tab)
- [ ] Browse Game Parameters (Game Design tab) and flag anything she wants to change

**Decide together:**

- [ ] Cadence — weekly? Twice a week? What day/time?
- [ ] Session 2 goal — she writes a full set of dialogue lines for one region + one emotion state, and Dave walks through how they load
- [ ] Communication channel between sessions — text? Discord? Shared doc?

---

## Session Principles

- **Her machine, her screen.** Let her drive as much as possible. Dave talks, she types.
- **Stop if it stops being fun.** This is a bonding project. If setup drags, skip to the dialogue edit — environment can finish async.
- **90 minutes is the sweet spot.** End on a high note, not exhaustion.
